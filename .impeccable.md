## Design Context

### Users
Solo developers and dev teams using LocalMail during local development and staging. The tool runs alongside their editor and terminal. Context: technical, focused, likely late evening or in a dark-themed environment.

### Brand Personality
Sharp · Precise · Minimal — like a precision instrument. Nothing wasted, everything intentional. The tool should feel at home next to VS Code and iTerm.

### Aesthetic Direction
Classic email client (Superhuman / Mimestream) — spatial, structured, elegant. Not a generic dev dashboard. The reading pane dominates; the list sidebar is dense and typographic. Dark theme throughout (dev context, terminal-adjacent). Warm amber `oklch(0.76 0.14 82)` as the single accent — editorial and unexpected for a dev tool.

### Design Principles
1. **Density without clutter** — pack information tightly but let hierarchy breathe
2. **Typography carries the weight** — color and decoration support, never lead
3. **The email content is the star** — chrome disappears once you're reading
4. **Every pixel earns its place** — if it doesn't add information or clarity, remove it
5. **Real mail client energy** — feel like Mimestream, not a UI component demo
